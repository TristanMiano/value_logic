### Author-intuition correction: what a positive ReLU value means

This correction is important enough to state independently of the architecture
details. For `z=ReLU(s)`, the sign has a four-level interpretation:

| stage or object | warranted reading of `z>0` | what remains unwarranted |
|---|---|---|
| arbitrary hidden preactivation | `s>0`, nothing more | adequacy, evidence, truth, or an interpretable feature |
| named learned margin `epsilon-J_hat` | predicted positive slack | actual threshold satisfaction or permission to rely |
| conservative margin `epsilon-upper(U_safe)` bound to accepted evidence | strict certificate-relative surplus for one atom, scope, and mode | target-world truth, other atoms, or the full profile license |
| complete exact profile plus active mask | the rectified surplus may be used by a declared downstream plan | authorization still comes from the exact profile/mask, not from `z` |

The zero branch is equally nonsemantic by itself. Inclusive support at equality
has `z=0`, as do negative, open, refuted, missing, invalid, and conflicted cases;
biases and bypass paths can still produce nonzero downstream output. Thus the
author's classifier intuition survives as a **named, staged dual-use surplus
construction**, not as the claim that ordinary positive activations are grants.
This distinction should be taught explicitly in both public artifacts because
the rejected interpretation is natural and therefore likely to recur.
Task 16 now supplies the canonical three-atom numerical/vector example that the
experiment and both public artifacts must reuse.

No completed task is reopened. The changes below are prospective integration,
experimental, and publication controls.

## 4. End-to-end implementable blueprint

The production data flow is:

```text
frozen request, finite state, and exact atom address
    -> whitelisted pre-outcome dependency projection
    -> learned statistic / payload / grade proposal
    -> proposal-bound calibration record
    -> exact evidence, scope, polarity, and checker gate
    -> accepted region or open obstacle
    -> exact boundary-aware K_3 atom diagnostic
    -> exact profile meet and public outcome
    -> exact profile active mask
    -> post-license router over active plans, or fallback
    -> machine-readable diagnostic and provenance trace
    -> independently ranked audit of the frozen whole system.
```

For a recursively structured plan, a parallel branch checks primitive evidence
and declared grade/certificate transformers, constructs a checked root
certificate, and then submits the root claim to the ordinary atom decoder. A
predicted grade plus a predicted interval is not that certificate.

This is implementable, but the repository intentionally does not yet contain
the end-to-end learner. Task 19 must freeze the following integration contract.

### 4.1 Bound accepted calibration to the exact proposal

The experiment must use one immutable accepted-envelope record that binds at
least:

```text
atom/statistic schema and candidate identity,
units and normalization identity,
scorer/configuration hash,
calibration procedure and split-manifest hash,
scope/group and alpha_cal,
eta_cal or an unbounded sentinel,
allowed evidence polarity,
checker and version,
validity interval,
provenance root.
```

The decoder rejects mismatches. An accepted checker record cannot be paired
with an unrelated scorer, proposal, radius, candidate, or scope.

### 4.2 Install a pre-outcome scorer firewall

The learned scorer receives only a declared whitelist. It may not receive the
target statistic, reference atom state, oracle route/cell label, target-derived
certificate region, decoded outcome, later audit result, or a record whose
contents trivially disclose one of them. Exact identity metadata may be
embedded only when its identity is preserved for the decoder and audit.

### 4.3 Use five blocked evidential roles

The minimum split is:

```text
train
envelope calibration
reject/router validation
system audit
final confirmation test.
```

Blocking is by independent generator world/trajectory, provenance root, and
plan family—not only by row identifier—so cloned atoms, time variants, or
multiple candidates from one latent instance cannot leak between roles.

### 4.4 Keep guarantee levels separate

Atomwise marginal conformal-style coverage is not profile-conjunction
coverage, selected-subset coverage, routed false-grant control, or system-level
adequacy. Each stronger object requires a separate audit estimate or a declared
simultaneous certificate. Exact zero inactive-selection error is an invariant
of masking, not an empirical achievement; router learning is judged on routed
loss, severity, coverage, and fallback use among eligible candidates.

### 4.5 Make dual use a staged interface

The safe dual-use construction is:

```text
proposal -> accepted envelope -> exact decoder
    -> <support bit, normalized positive surplus,
        diagnostic, evidence handles>
    -> named downstream consumer.
```

It is not an arbitrary hidden activation in one uninterrupted MLP. Supported
equality has zero surplus, so the exact support bit remains indispensable.

### 4.6 Compare the objective fairly

The structured and atom-cross-entropy arms share the same allowed input,
including tolerance where relevant, and the same exact `WF`, missingness,
validity, polarity, and evidence gates. Cross-entropy predicts the remaining
`K_3` relation; it is not handicapped by having to learn exact facts supplied
symbolically to the structured arm. Labels come only from the generator-fixed
oracle/evidence record. Tolerance-transfer fidelity, false support/refutation,
and boundary risk at matched coverage are common endpoints. Structured-region
coverage and class-probability calibration are different measurements and are
not collapsed into one “calibration” number.

### 4.7 Separate system certification from confirmation

Freeze `e_sys`; build its candidate certificate from the system-audit split at
a lower evidential rank; assess it under independently fixed `q_sys`; and then
test the frozen assessment on the untouched confirmation set. The
self-confidence arm is an explicitly invalid pseudo-certificate comparison,
never another accepted evidence mode.

## 5. Minimum publishable empirical core

The original experiment block was at risk of becoming an omnibus benchmark
with three underpowered headings. The minimum empirical result is now fixed
prospectively:

1. one reusable synthetic succession fixture with `Granted`, `Withheld`,
   `Refused`, and `Undefined`; supported/open/refuted atoms; valid, invalid, and
   missing evidence; simultaneous licenses; an empty active set and fallback;
   lapse versus rebuttal; relevant versus irrelevant updates; changed
   tolerance; and a later finite-set dominator;
2. `F35/H18.1`: standardized center--radius statistic learning plus accepted
   held-out expansion versus independent atom cross-entropy on tolerance
   transfer and boundary fidelity;
3. `F36/H18.2`: absolute marginal coverage and width at the exact registered
   exchangeable scope, without renaming it selected/profile/system coverage;
4. false support, false refutation, atom fidelity, and query-quotient fidelity
   after the same exact decoder;
5. an in-regime noninferiority guard so transfer gains cannot hide unacceptable
   ordinary degradation; and
6. required ablations 1--3 and 5 from Task 18: cross-entropy, center-only,
   predicted radius without accepted held-out expansion, and self-confidence
   incorrectly treated as a grant signal.

The generator cannot be frozen until Task 19 names numerical superiority and
noninferiority margins, the primary endpoint family and multiplicity rule, the
unit of replication, and the planned uncertainty/test rule. Task 19A uses only
a disjoint pilot seed bank to validate boundary prevalence, state balance,
variance, runtime, and sample size. Final worlds remain embargoed.

Independent generator worlds or trajectories are the statistical units.
Atoms, candidates, and time slices within a world are clustered observations;
training seeds are repeated fitting attempts, not independent target worlds.
Boundary cases receive positive design mass, with stratum-specific results and
results reweighted to the declared target distribution both reported.

The seam/routing and certificate/system tiers are separate extensions. Their
absence does not falsify a claim, and a deterministic integration stress suite
must not be marketed as a powered empirical comparison.

## 6. Comparator decision

### 6.1 Decision

The minimum core uses **no trained non-ReLU architecture comparator**. Its
finite statistical budget goes to `F35/H18.1`, registered calibration, and the
grounded-system integration boundary.

**Hard mixture-of-experts is the sole eligible optional comparator.** It may be
activated in the routing/seam extension only after Task 19A shows that the
architecture-by-seam interaction is separately powered. If it fails that
gate, it remains a nominated follow-up against the frozen generator. This is a
prospective omission, not an inconclusive result.

The comparator is architecture-level: matched ReLU expert subnetworks are
allowed inside the MoE; the tested difference is the hard learned gate that
can create a discontinuous assembly.

### 6.2 Why this comparator and not the others

- A hard MoE directly addresses Task 17's trace-agreement/discontinuity
  theorem. The mathematics predicts the exact family on which an ordinary
  continuous output cannot realize the target.
- A monotone/lattice model would mainly duplicate tolerance monotonicity that
  the exact decoder already guarantees. No other sufficiently important
  learned monotone coordinate is currently selected.
- A graph/set model has a valid `F33` hypothesis, but two illustrative DAGs do
  not power structural generalization. It is deferred until a study contains
  many independent typed graph families and true structural separators.

### 6.3 Fair hard-MoE contract

If the gate passes, both architectures receive the same nonleaking inputs and
emit the same center, radius, payload, and grade schemas. They use the same
worlds, training updates, normalization, objective, tuning-trial budget,
symbolic decoder, active mask, post-license router, and seed schedule. Each
fits its own identically specified calibration record. Total trainable
parameters are the primary capacity match; active parameters, training and
inference FLOPs, wall time, and tuning count are reported separately.

The learned MoE gate receives no oracle regime label. An oracle gate is allowed
only as a labeled expressivity upper bound. The study uses paired conforming
and mismatched seam families, with independent seams as the statistical units.
A finite continuous network can fit finitely many sampled points with a steep
transition, so ordinary average error is insufficient: near-seam
extrapolation, maximum/quantile error, transition width or Lipschitz behavior,
and the architecture-by-seam interaction must be prespecified. The theorem,
not a finite test, carries the exact impossibility claim.

## 7. Provisional narrative contract for the public artifacts

The repository can now answer the checkpoint's lead question, but this answer
is provisional until Task 25 freezes it after the experiment.

> Physics motivates a decision problem before it settles a metaphysical one.
> A later theory can displace an earlier theory as a universal account while
> leaving the earlier theory reliable enough for a restricted domain, purpose,
> and resource budget. A bounded agent therefore needs more than the labels
> “true” and “false.” It needs a revisable record of whether the current
> evidence supports reliance, supports a counterclaim, or leaves the matter
> unresolved; it must also remember overlaps, gaps, alternatives, and what was
> actually searched.
>
> This motivates value logic's basic design. Model and domain come directly
> from theory succession. Loss and `epsilon` do not. They enter when “still
> useful” is turned into an operational rule for action. Sometimes a tolerance
> is imposed externally. Sometimes the status quo supplies it: if fallback
> loss is `J_q(F_q)` and changing course must improve by `Delta`, then the
> largest acceptable candidate loss is
> `epsilon_F=J_q(F_q)-Delta`. The resulting judgment is not a degree of truth.
> It is a finite-stage, evidence- and profile-indexed license that can be
> granted, withheld, refused, or undefined, revised by new evidence, and held
> simultaneously for several overlapping plans.
>
> Why expect this design to fit a neural network at all? The compatibility was
> engineered at a typed boundary and then checked mathematically. Quantitative
> atom statistics, signed margins, and proof-erased payload/grade computations
> are separated from exact identity, evidence validity, boundary convention,
> open status, certificates, active masking, and fallback. When the numerical
> map is global finite continuous piecewise affine, Task 17 proves that a
> finite ReLU network realizes it exactly. A named normalized ReLU value can
> then have a genuine dual use, but only by construction. An arbitrary positive
> activation means only positive preactivation; a named learned margin means
> predicted slack. After an accepted conservative envelope and exact decoder
> establish the relevant atom, positive activation can mean strict
> certificate-relative surplus and serve as numerical input to a later declared
> computation. The full profile license still comes from exact state and mask.
>
> The limits are as important as the construction. ReLU zero merges supported
> equality with failure and uncertainty. Equal adequacy values can hide
> different payloads. Units can change magnitudes. A continuous ReLU output
> cannot realize a mismatched hard seam. A fixed output registry does not know
> every future model. Most importantly, a neural score cannot validate its own
> evidence. The result is therefore not “logic is an MLP.” It is a hybrid
> architecture with a proved continuous realization, exact symbolic and
> evidential boundaries, and empirical questions about learnability,
> calibration, routing, and transparency that remain to be tested.

This is more accurate than saying “everything continuous is CPWL.” The exact
theorem applies only when a global finite CPWL map or extension is actually
specified. Nor should the story imply that the correspondence was inevitable:
the interface was designed to expose a tractable numerical part, the theorems
then discovered sharp success and obstruction conditions, and the experiment
tests whether the designed factorization learns usefully.

### 7.1 Reader-facing order

Both public artifacts should proceed:

```text
physics succession
    -> why reliance precedes final truth
    -> fallback/status quo and the origin of epsilon
    -> finite-stage licenses, overlap, gaps, and revision
    -> classifier/dual-use ReLU intuition
    -> the hybrid realization and its boundaries
    -> experiment, recursive plans/system audit, and limitations.
```

This deliberately changes Fable's suggested Substack order by putting
fallback-derived `epsilon` before the classifier example. The project author
asked for the origin of error/tolerance to appear early, and the logic is more
motivated when the threshold exists before its neural encoding.

### 7.2 Main text versus repository/appendix

The main paper should retain the four motivating questions, one running
succession example, the compact three-carrier calculus, roughly three to five
headline contributions, the strongest countermodels, the neural factorization
and walls, and the minimum empirical core. Full record schemas, reason
taxonomies, audit IDs, test inventories, optimizer schedules, exhaustive
ablations, and routine quotient lemmas belong in appendices or repository
documentation. Theorem 7's coordinate-complete minimality is a scoped
formalization, not a headline; the equal-code separator, boundary collision,
normalization, and positive construction carry its reader-facing significance.

The Substack version should use at most one displayed formal judgment and
should explain the same distinctions through the running example rather than
through the repository's control vocabulary.

## 8. Difficulty, failures, successes, and discoveries

### 8.1 Harder than predicted

Task 14B was harder than the previous audit forecast because the proposed
present-history dependency lemma was itself insufficient. A future
certificate, correction, or dominator is absent from present provenance but
can change an atom. Treating absence as a read of a collection-index key
repaired the specification, not merely the proof. The checkpoint gate worked:
it accepted the problem, rejected the auditor's inadequate construction, and
produced a stronger phantom-safe locality theorem.

### 8.2 As difficult as predicted

Task 17 benefited from freezing its seven target statements before choosing
the final encodings. Every target was proved or sharply delimited without
architecture-driven theorem drift. The dual-use minimality clause needed the
positioning correction now applied, but its formal statement is sound.

### 8.3 Easier or faster than predicted

Tasks 15--18 were specified and verified unusually quickly. Fable's independent
rederivation found no mathematical error, the theorem-to-encoding mapping held,
and the executable witnesses are substantive. This says that the specification
and proof phase moved faster than expected; it says nothing yet about training
stability, runtime, sample complexity, or empirical success. The experiment
gate is therefore the right place to slow down.

### 8.4 Failures

No failure was silent. The productive formal falsification was `F18`: zeroing
one ReLU channel does not quarantine a network with biases or bypass paths.
Its complementary semantic correction is that positivity alone does not grant:
meaning progresses from arbitrary sign, to predicted margin, to accepted
certificate-relative atom surplus, and only then to exact profile licensing.
The project impact is propagated to exact masking, required failure witnesses,
and the public narrative. Fable's earlier S6 proposal was also a specification
failure, now recorded as such. No central claim in Tasks 14B--18 was otherwise
falsified.

### 8.5 Discoveries to surface

1. the tight factor-of-two conservative recovery band;
2. the boundary collision at zero, which disproves scalar score-as-status;
3. `epsilon_F=J_q(F_q)-Delta` as an operational origin of tolerance;
4. absence-as-readable-key locality, with its database ancestry;
5. the seam trace-agreement characterization and discontinuity obstruction;
6. conditional survival of the author's dual-use activation intuition; and
7. the paired result that finite recursive plans can carry checked evidence
   while a system cannot certify itself solely by self-endorsement.

## 11. Remaining risks

1. **Target leakage:** the dependency record can contain target-bearing fields
   unless the whitelist is frozen and tested.
2. **Certificate substitution:** an accepted gate can be misbound to another
   proposal unless the immutable record contract is implemented.
3. **Pseudo-replication:** atom rows and training seeds can falsely inflate
   power unless worlds/trajectories are the units.
4. **Guarantee inflation:** marginal atom coverage can be mislabeled as
   profile, selection, routed, or system safety.
5. **Synthetic overreach:** the running example is Newtonian-like only as a
   narrative structure; it is not evidence about historical physics.
6. **Architecture non-test:** if both ReLU and MoE simply use the same external
   hard router, the comparison does not test the seam theorem.
7. **Register inversion:** technically correct audit prose can still make an
   unreadable paper. Task 25, Checkpoint D, and Task 31A now guard this.
8. **Optional-branch sprawl:** routing, system, graph, and policy/value work
   must not dilute the minimum core or be called confirmatory without power.
