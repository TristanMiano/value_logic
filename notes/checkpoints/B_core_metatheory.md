The compact calculus is coherent enough to support a neural correspondence, and its mathematical center is no longer a record schema. The paper-level object remains the request `(s,e,q,P)` over the three principal carriers `E,Q,S`, with `W` semantic, `WF` separate from meaningful `K_3` atom values, and profiles supplying finite consumer queries. The quantitative atlas/router machinery remains an extension.

The theorem bar is nevertheless only conditionally cleared. The profile soundness/relative-completeness cluster is a genuine logic result. The open-ended stability package is usable after small precision fixes and explicit attribution of its classical indistinguishability and sequential-decision patterns. The update-persistence characterization does not yet count as a theorem of the concrete calculus: its graph assumptions currently stipulate accurate influence without proving that the frozen atom clauses induce such a graph. A new Task 14B will close exactly this gap and repair the executable/reference-level defects found by the audit. No broader metatheory expansion is commissioned.

The neural target is now architecture-neutral and smaller than the complete formalism. A learned component should supply dependency-scoped atom sufficient statistics and calibrated uncertainty; exact metadata, `WF`, profile roles, `K_3` aggregation, active-set masking, fallback, and provenance identity normally remain symbolic or external. ReLU remains the reference witness for continuous piecewise-linear statistics. Two alternatives are nominated only as hypotheses: a monotone/lattice model for selected monotonicities and a hard mixture-of-experts router for discontinuous seams. Graph/set architectures are deferred unless Task 15 makes variable-library or plan-DAG structure an experimental hypothesis.

Task 17 is the next prospective mathematical centerpiece. Its target statements are fixed below before Tasks 15–16 choose encodings.

## What the neural layer must and need not represent

The semantic interface should factor as:

```text
typed request and registry data
    -> atom address plus dependency-scoped input x(s,e,q,a)
    -> learned or verified sufficient statistics Z_a
    -> fixed atom decoder and indexed diagnostic
    -> profile query and K_3 meet
    -> four-way outcome, active set, selection or fallback.
```

### Required learned or estimated quantities

Only quantities not already mechanically available should be learned. Depending on the selected fragment, these may include:

- risk, regret, cost, robustness, or certificate-region estimates;
- calibrated error regions or validity estimates for empirical certificates;
- pairwise comparison statistics when they cannot be computed from certified records;
- selection scores after the licensed mask is applied; and
- theorem-indexed extension quantities such as selected-scope risk, misroute severity, bridge disagreement, or plan-component sensitivities.

For a scalar certificate region `[L,U]` and tolerance `epsilon`, a natural pair is

```text
support slack = epsilon - U,
refute slack  = L - epsilon.
```

The external decoder supports when the first slack is nonnegative, refutes when the second is positive, and otherwise leaves the atom open. Missingness, invalidity, conflict, and scope mismatch stay explicit; they are not negative margins. If estimated statistics have a certified error `delta`, the safe interval is enlarged before decoding, and the boundary band is conservatively open.

### Symbolic or external structure

The default design keeps the following outside the learned scorer:

- `WF` and `WFDiag` when type/denotation/executability are mechanically decidable;
- atom address, exact scope, loss/criterion, certificate mode, profile slot, and safety/report role;
- profile instantiation and finite `K_3` meet;
- evaluated-set/search identity and exact trace-verifier results;
- provenance graph and registry identifiers;
- active-set masking, the ban on selecting an unlicensed plan, and empty-set fallback;
- library expansion and model/version storage; and
- direct composite certification or a symbolic Task 14A propagation calculation.

This division is a default, not a ban on learned validity or routing. When such a quantity is learned, it becomes a new evidence-bearing model with calibration and provenance rather than an invisible replacement for the semantic field.

### Consumer-relative compression

For a fixed profile query family `F`, exact status prediction needs to separate the classes of `V / ~_F` and the required well-formedness observations. A full audit interface is finer: it preserves address-indexed witnesses, counterwitnesses, obstacles, safety projections, and provenance pointers, as well as possible future profile queries.

Two qualifications are now mandatory:

1. `ceil(log_2 |V / ~_F|)` is a lower bound for a finite discrete code, not a lower bound on neural output width. One unrestricted real coordinate can name finitely many classes unless precision, robustness, noise, or decoder restrictions are imposed.
2. `Ill(WFDiag)` versus `Well([v]_F)` is a canonical typed normal form. An arbitrary sufficient code need only decode to, or refine, that quotient; it need not literally use a tagged datatype internally.

The coordinate-indispensability theorem similarly says that the assessment must remain sensitive to distinctions carried by `s,e,q,P`; it does not require four neural modules or four embedding blocks.

## Task 17 target statements frozen by this checkpoint

Tasks 15 and 16 must design toward the following theorem statements rather than choosing convenient encodings first.

1. **Architecture-neutral exact factorization.** For a declared input class and profile family `F`, an encoder/scorer plus deterministic decoder realizes every requested status exactly only if equal codes imply equal required `WF` observations and equal `V / ~_F` classes. Conversely, any code preserving those observations is status-sufficient. Diagnostic-preserving clients require the corresponding finer quotient.

2. **Margin-robust approximate realization.** If every learned continuous sufficient statistic has uniform error at most `delta`, then requests separated from all relevant support/refutation boundaries by more than the propagated error decode exactly; requests inside the uncertainty band are conservatively open. The result must state how multiple statistics, profiles, and certificate modes propagate error.

3. **Finite ReLU construction.** On a fixed finite-dimensional encoding, if the required real-valued atom statistics are continuous piecewise linear, a finite ReLU network with affine outputs computes them exactly under stated architecture, size/depth, and domain conventions. A fixed external, boundary-aware `WF + K_3` decoder then recovers the selected finite profile queries. Direct discrete `K_3` output and inclusive boundary decisions are not themselves continuous ReLU outputs.

4. **Polyhedral hard-seam characterization.** For affine experts on a conforming finite polyhedral complex, the assembled map is continuous exactly when expert traces agree on every relevant nonempty common face/intersection. Under that condition it is CPWL and receives an exact finite ReLU representation; disagreement on an accumulation face makes the hard assembly discontinuous and impossible for one ordinary continuous ReLU output, while an external discrete router or hard mixture of experts may still represent it.

5. **Finite versus expandable library limitation.** A fixed model-indexed output supports only its declared finite registry. An expandable design uses candidate-conditioned shared scoring, sparse set/pair reduction, and external identity/selection. Neither design converts evaluated-set non-domination into global closure.

These are targets, not results already proved. Task 17 must verify the exact CPWL representation hypotheses, global-domain/extension convention, boundary cases, and size/depth attribution from primary sources.

## Architecture selection

Alternatives were screened by five criteria: preservation of the same semantic interface; a distinct theorem or empirical hypothesis; relevance to a surviving calculus property; matched-capacity/compute feasibility; and a failure criterion that is not merely “different accuracy.”

| architecture | nominated question | present decision |
|---|---|---|
| ReLU MLP plus symbolic decoder | Exact continuous-CPWL statistic realization, inspectable threshold margins, and activation geometry | **Reference architecture** |
| Monotone/lattice model | Does enforcing selected tolerance/refinement monotonicities improve extrapolation or calibration compared with learning them? | **Candidate comparator**, limited to coordinates whose monotonicity is actually proved |
| Hard mixture of experts / external hard router | Does native discontinuous routing improve seam-mismatch cases while preserving license masking and fallback? | **Candidate comparator** and direct foil for Task 17 |
| Neuro-symbolic decoder | Can exact `WF + K_3` aggregation prevent inconsistent aggregate status? | **Shared infrastructure**, not a comparator |
| Graph/set model | Does explicit permutation/DAG structure improve variable-library or composed-plan generalization? | Deferred until Task 15 establishes a specific hypothesis |
| Maxout | Another continuous piecewise-linear family | Not selected: currently redundant with the ReLU representation question |

Checkpoint C may select at most one trained comparator for the toy experiment unless two distinct, adequately powered hypotheses survive. Architectural monotonicity and hard routing must not be advertised as general interpretability improvements without evidence.

## Mathematical economy

Checkpoint B adds no new principal carrier, truth value, reason enum, bridge type, or license predicate. The typed record inventory remains an elaboration. Operational atom families are instances of a finite typed atom set and valuation, not a growing ontology for the paper. `RelUndom` and `CertUndom` may be presented as modes of one comparison atom when doing so preserves their different evidence conditions. Diagnostics remain one atom-indexed dependent sum over `K_3`; human labels remain renderings.

Task 14B's query/read and event/write footprints are derived interfaces for the operational clauses, not new paper-level sorts. Current provenance nodes explain an assessment; typed key footprints determine which future writes could change it. The two should not be conflated.

The profile/query quotient may remove redundant status distinctions for a fixed consumer. Support-entailment can also remove redundant requirements when only grant preservation is sought, but it need not preserve `Refused` versus `Withheld` or the original diagnostic explanation. Any such normalization must keep the unquotiented address map for auditability. It is therefore an encoding option, not a new primitive definition.

## Risks after this checkpoint

1. Task 14B could become another schema exercise. Its success condition is a case-analysis theorem connecting actual atom reads and event writes to change-completeness, not a longer dependency record.
2. Task 17 could prove only that ReLU networks are universal approximators. That would not meet the frozen exact/robust/seam targets.
3. The query quotient could be misread as permission to discard diagnostics or as a neural-width lower bound. Both readings are now explicitly prohibited.
4. The experiment could become an omnibus benchmark too small to test any claim well. The tiered design is now required.
5. The two comparator candidates could create breadth without power. Checkpoint C must select by a concrete hypothesis and fairness budget.
6. The project may still overstate classical stability patterns or standard transport mathematics. Task 14B and Task 31 own explicit attribution/classification checks.
